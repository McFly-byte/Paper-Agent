以下方案以“生产级多 Agent 学术调研系统”为目标设计，核心原则：**强推理主模型 + 结构化约束 + 分层调用（router）+ 显式抗幻觉机制**。
你当前有 **qwen3.5:122b**，这是一个非常好的“中枢模型”，因为该系列具备长上下文（≈256K）与较强推理能力，适合复杂分析与写作 ([ComputingForGeeks][1])。

---

# A. 各节点模型选型（主 + 备选）

## 1. reading_node（文献清洗 / 摘要 / 信息抽取）

**主模型：**

* **Qwen3.5 122B（非 thinking 模式）**

**备选：**

* Qwen3.5 27B（降本/并发）
* DeepSeek-V3 / DeepSeek-R1-distill（更偏结构化抽取）

**理由：**

* reading 是“弱推理 + 强结构化”任务
* Qwen 系列在多语言、信息抽取上稳定性强 ([Mindplix][2])
* 不需要 full reasoning（否则成本浪费 + latency 增大）

---

## 2. analyse_node（聚类 / 深度分析 / 全局洞察）

**主模型：**

* **Qwen3.5 122B（thinking 模式开启）**

**备选：**

* DeepSeek-R1（强推理专用）
* Mixtral 8x22B（性价比 fallback）

**理由：**

* analyse 是系统“认知中枢”
* 需要：

  * 跨文献归纳
  * 研究脉络抽象
  * gap 发现
* DeepSeek-R1 类模型在 reasoning 上表现突出 ([simplilearn.com][3])

---

## 3. writing_node（大纲 + 分节写作 + RAG）

拆开看（非常关键）：

### (1) 大纲生成

**主模型：**

* Qwen3.5 122B（thinking）

**备选：**

* DeepSeek-R1
* Llama 3 70B

**理由：**

* 需要全局结构规划（类似 planning agent）

---

### (2) 分节写作（正文生成）

**主模型：**

* Qwen3.5 122B（非 thinking 或低 thinking）

**备选：**

* Qwen3.5 27B（高并发写段落）
* Llama 3 70B（稳定写作风格） ([superallen.org][4])

**理由：**

* 写作是“弱推理 + 强语言能力”
* 可用 smaller model 提升吞吐（典型优化点）

---

### (3) 检索增强写作（RAG fusion）

**主模型：**

* Qwen3.5 122B（thinking）

**备选：**

* DeepSeek-V3
* Mixtral 8x22B

**理由：**

* 需要：

  * 引文融合
  * 去冲突
  * 防 hallucination

---

## 4. report_node（最终整合 / 格式化 / 输出）

**主模型：**

* Qwen3.5 122B（non-thinking）

**备选：**

* Qwen3.5 27B
* Gemma 3 27B

**理由：**

* 偏工程化任务：

  * 格式统一
  * Markdown/LaTeX 输出
* 不需要 heavy reasoning

---

# B. 各节点推荐参数

| 节点          | temperature | top_p | max_tokens | repetition_penalty |
| ----------- | ----------- | ----- | ---------- | ------------------ |
| reading     | 0.2         | 0.8   | 2k–4k      | 1.1                |
| analyse     | 0.5         | 0.9   | 6k–12k     | 1.2                |
| outline     | 0.4         | 0.9   | 4k–6k      | 1.1                |
| writing（段落） | 0.6         | 0.95  | 2k–4k      | 1.05               |
| RAG写作       | 0.3         | 0.85  | 4k–8k      | 1.15               |
| report      | 0.2         | 0.8   | 6k–10k     | 1.1                |

补充原则：

* **结构化任务 → 低 temperature**
* **生成性任务 → 提高 temperature**
* **长文本 → 提高 repetition_penalty 防啰嗦**

# C. 单模型 vs 分层（Router策略）

结论：**必须分层（强烈建议）**

推荐架构：

```
             ┌──────────────┐
             │ Router Node  │
             └─────┬────────┘
                   │
      ┌────────────┼────────────┐
      ↓            ↓            ↓
 small model   mid model   strong model
 (27B)         (70B)       (122B)
```

### 路由规则（实用级）：

| 条件                | 模型   |
| ----------------- | ---- |
| token < 2k + 简单抽取 | 27B  |
| 单文档分析             | 70B  |
| 多文档归纳 / 写作规划      | 122B |

# D. 风险点与缓解策略（核心）

## 1. 幻觉（最大风险）

**高发节点：**

* analyse_node
* writing_node（尤其RAG）

**解决方案：**

* 强制 citation schema：

  ```
  {claim, evidence_doc_id, confidence}
  ```
* RAG写作改为：

  * “先抽取证据 → 再写”

---

## 2. 格式漂移

**表现：**

* JSON 崩
* Markdown结构错乱

**解决：**

* function calling / schema约束
* 二阶段：

  ```
  step1: 内容生成
  step2: 格式修复（小模型）
  ```

---

## 3. 长上下文截断

问题来源：

* Ollama KV cache + context limit

**策略：**

* sliding window + memory summary
* analyse_node 输入做：

  ```
  chunk → 局部summary → 全局summary
  ```

---

## 4. Agent 工具调用失败

**表现：**

* search结果没被消费
* reading失败

**策略：**

* Tool result 强制结构：

  ```
  {title, abstract, chunks, embedding}
  ```
* 所有 node 输入标准化

---

## 5. 推理过度（thinking模式副作用）

**表现：**

* latency爆炸
* 输出冗长

**策略：**

* 仅 analyse / outline 开启 thinking
* writing 默认关闭

# 关键架构建议（超出问题但强相关）

1. **把 writing_node 拆成 pipeline（非常关键）**

   ```
   outline → section planner → section writer → reviewer
   ```

2. **增加 reviewer_node（强烈建议）**

   * 模型：Qwen3.5 122B
   * 职责：

     * 检查 hallucination
     * 检查逻辑一致性

3. **引入“证据优先流”**

   ```
   search → extract → evidence pool → analyse → writing
   ```

---

# 不确定性说明（重要）

* Qwen3.5 122B 在 Ollama 的 **thinking 模式细节实现**（开关粒度）存在差异（不同量化/版本）
* DeepSeek-R1 本地部署版本性能差异较大（distill vs full）
* 超长上下文（>128K）在实际推理中会出现性能/质量下降（业界共性问题）

---

# 一句话结论

你的系统已经具备雏形，下一步提升不在“换模型”，而在：

> **用 Qwen3.5 122B 做“中枢推理”，用小模型做“执行层”，用结构化约束解决幻觉问题。**
